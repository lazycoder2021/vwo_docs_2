Different Types of VWO SmartCode

  • Updated
  • November 03, 2021 09:40
Share:

Follow
NOTE:   You can skip this article and jump to How to Configure VWO SmartCode? if you are not really interested in knowing about the different types of SmartCode. It is enough if you know that the default SmartCode option suggested by VWO is Asynchronous SmartCode (and that you'll be facing issues if you don't select the same). But, if you like tech stuff, read on to know more.

Web page loading time can have a critical impact on website conversions. And with third-party tools (like VWO, for example) connected to your website, the loading time can get affected. With this in mind, VWO makers came up with Asynchronous SmartCode, which ensures your website's loading time is not affected, as VWO works behind the scenes to increase your website conversions.

What is Asynchronous SmartCode? Why is it so effective?

  • Web page loading happens in sequential order - from the top (meta tags in the head section) to the bottom (closing of the body tag).
  • So if any script running on your page (especially the ones placed in your head tag) causes a delay, this affects the entire web page's loading time.
  • In fact, sometimes the web page might not load at all.

This web page loading-related issue does not happen with asynchronous smartcode.

  • Because asynchronous smartcode does not interrupt the sequence of web page loading -- just because there is a delay in reading the smartcode by VWO servers (maybe because of poor connectivity issues).
  • Asynchronous smartcode has the ability to let the web page load first and then connect with the VWO servers after that, thus preventing critical loading-related delays if any.

Why You Shouldn't Use Synchronous SmartCode

  • Because they depend on the sequential loading of the web page. This means if there is a delay in running the smartcode script, this will affect the web page load time greatly.
  • So if you are living in some remote location where there are frequent connectivity issues, then chances are some of your VWO campaigns might not run at all - just because you are making use of synchronous smartcode! So always make use of asynchronous smartcode.
